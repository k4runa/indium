DOCS // getting started

Zero to a running game.

This guide takes you from an empty folder to a standalone, shippable build: install Indium, learn the editor, write a hot-reloaded C++ script, export.

1 · Install

Grab the latest release for your platform from the download page or GitHub Releases. On macOS, open the .dmg and drag Indium to Applications; on Windows/Linux, extract the archive anywhere and run the Indium executable.

ℹ
On Windows the zip bundles a portable C++ toolchain and the script SDK — hot-reloaded scripting works without installing MSYS2 or Visual Studio. On macOS the .app is ad-hoc signed: clear the download quarantine once with xattr -dr com.apple.quarantine /Applications/Indium.app, and install the Xcode Command Line Tools (xcode-select --install) so clang++ is available for script compilation. On Linux you need a C++20 compiler (g++ or clang++) on your PATH.

Building from source

You need CMake ≥ 3.20, a C++20 compiler, and raylib 5.5 installed:

git clone https://github.com/k4runa/indium
cd indium
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
./build/Indium

2 · Create a project

Indium opens on the Hub — your project launcher. Click New Project, pick a name and a location, and hit Create. The engine scaffolds the project structure and compiles the template script:

MyNewGame/
├── project.indp        # project manifest
├── Scenes/main.scene   # your first scene
├── scripts/            # C++ gameplay scripts (hot-reloaded)
├── Assets/             # sprites, audio, fonts
├── dialogue/  cutscenes/  quests/  items/
└── Settings/

Existing projects appear as cards on the Hub — click one to open it, right-click for management actions like reveal-in-explorer or delete.

3 · The editor at a glance

The editor is a dockable workspace: Hierarchy (scene tree) on the left, the Viewport in the center, the Inspector (selected entity's components) on the right, and the Console / Content Browser at the bottom. Drag panel edges to re-arrange — the viewport keeps the world fixed while you resize.

4 · Your first entity

Right-click empty viewport space → Create. Pick a Rectangle, Circle, Sprite, Text, Light 2D, Tilemap, a Camera and more. The entity spawns where you right-clicked, already selected.

With it selected, the Inspector shows its Transform, color, and components. Use Add Component to attach behavior — a RigidbodyComponent for physics, a SpriteRenderer for textures, an AnimatorComponent for sprite-sheet animation, or one of your own scripts.

★
Drop a Light 2D into a scene and lighting "just works" — the editor activates the light pass automatically, with soft shadows and an ambient color you can tune in Project Settings.

5 · Your first script

Create scripts/Coin.cpp in your project (any editor — the engine generates .clangd and VS Code IntelliSense config for you):

#include "IndiumEngine.hpp"
using namespace Indium;

class Coin : public NativeScript
{
    IND_PROP(float, spin, 90.0f);     // degrees per second — edit it in the Inspector

    REGISTER_SCRIPT(Coin)

    void OnUpdate(float dt) override
    {
        entity->rotation += spin * dt;
    }

    void OnTriggerEnter2D(Entity* other) override
    {
        if (other->tag != "Player") return;
        auto& story = StoryState::Get();     // narrative blackboard
        story.Set("coins", story.GetInt("coins") + 1);
        Destroy(entity);
    }
};

INDIUM_EXPORT_SCRIPTS(Coin)

Back in the editor: Scripts → Compile & Reload. The compile runs in the background; when it lands, your scene is rehydrated against the new code — selection and state preserved. Attach the script to an entity via Add Component, and spin appears as an editable, serialized field in the Inspector.

ℹ
Scripts are plain C++20 — the full engine API (reference), raylib, and the standard library are all directly available. No bindings, no wrappers.

6 · Play, scenes & saving

Press ▶ Play. The scene is snapshotted, scripts get their OnAwake / OnStart, and the game runs in the viewport (switch the viewport's Game tab to see through the game camera). Press ⏹ Stop and the scene snaps back exactly to its pre-Play state — experiment freely.

Scenes live under Scenes/; manage them from the Scenes menu and switch at runtime from a script with LoadScene("level2"). Save slots, autosave on scene switches, and checkpoint zones are built in — see SaveManager.

7 · Export your game

File → Export Game packages your project together with the standalone IndiumPlayer runtime into a distributable folder — your scenes, assets, compiled scripts and settings, no editor attached. Zip it, ship it.
